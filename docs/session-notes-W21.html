5. Key Learnings & Fixes

IssueFix / Rule Going Forward
Adult Images not found in root folder IMPORTANT Photos live in Image Resources subfolder (1lhVTu2tZ9Tgs3ZYFZnTC1MqJApldro1K), not the root Adult Images folder. templates.json now points directly to subfolder.
Kids Images files have UUID names (no descriptions) Pick by file size — larger = better quality. Target >400 KB. Do not try to read meaning from the filename.
Training Videos (Kids) appeared empty in first audit Folder is NOT empty. Re-run search if it appears empty. Contains MP4 files — prefer MP4 over MOV.
Canva merge-designs multi-operation error ONE operation per merge call. For 4-slide carousel: 1 create_new_design call + 3 modify_existing_design calls = 4 sequential calls total.
Drive URL for videos (>25 MB) broke Use drive.usercontent.google.com/download?id=X&export=download&authuser=0&confirm=t for all videos. Never use drive.google.com/uc?id=X.
lh3 URL used for video → Canva stored a poster frame image lh3 pattern is for images ONLY. If upload response shows "type":"image" but source was a video, the wrong URL was used.
GitHub Pages 404 after push Repo must be public AND Pages must be explicitly enabled. Enabled via: POST /repos/jeffd1130/Alliance_Cobrinha_LA/pages with {"source":{"branch":"main","path":"/docs"}}.
Auto-push hook not firing during session Hook in project's .claude/settings.json only loads when Claude Code is opened FROM that project directory. When opening from a different dir, push manually.
